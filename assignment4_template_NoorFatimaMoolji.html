Research Question

My research question is:

“Has Melbourne’s public transport network fully recovered from COVID-19, and what does the pattern of recovery reveal about how commuter behavior has permanently changed?”

Why I chose this question:

I first explored Melbourne by train during a month-long visit before COVID. It was busy, reliable, felt like a city that ran on its public transport. Returning later, the same trains felt noticeably emptier, particularly at the start and end of the week. That observation stuck with me and became the seed of this analysis.

Sub-questions:

  1. Which PT modes have recovered to pre-COVID levels and which haven’t?
  2. Is the recovery gap driven specifically by weekday commuter travel?
  3. Which days of the week show the weakest recovery and does this suggest a permanent shift toward hybrid work?
  4. Does the persistence of the patronage gap despite rising fuel costs suggest the shift is structural rather than price-driven?

Data Description

Dataset 1: Monthly Average Patronage by Day Type and by Mode

Source: Department of Transport and Planning, Victoria
Portal: data.vic.gov.au
URL: https://discover.data.vic.gov.au/dataset/monthly-average-patronage-by-day-type-and-by-mode
Licence: Creative Commons Attribution 4.0 International
Last accessed: 7th June 2026

Type of data:

This is administrative sample data not a census of all trips. As stated in the dataset documentation:

“Patronage is a count of patron boarding onto services with the exclusion of train transfers. Patrons includes all persons 5 and over, excluding drivers and station staff.”

“Patronage data is derived from myki ticketing data. As ticketing data provides an incomplete picture of the number of people using public transport, DTP also conducts a patronage survey to supplement myki data. The purpose of the patronage survey is to determine the transaction rate, which is the percentage of passengers who ‘touch-on’ when they travel. Ticketing transactions are boosted according to the transaction rate to provide an estimate of total patronage.”

This means every figure in the data set is an estimate of total boardings, not a direct count.

Data quality notes from the documentation:

  • Patronage estimates are rounded to the nearest 50; values less than 50 are rounded up to 50

  • As of January 2021, metropolitan train patronage uses barrier count data where available, in place of survey observations, which means the methodology changed mid-series, which should be noted when interpreting 2020 vs 2021 figures

  • From 2021 onwards, regional train data includes non-myki patronage (e.g. paper ticket counts) prior to this it does not, meaning regional train figures before and after 2021 are not directly comparable

  • Regional bus data excludes services that are not myki-enabled

  • Data is updated monthly with a 2-month lag

Ethical considerations: This dataset contains no individual-level data. All figures are pre-aggregated to monthly averages so there is no information about individual passengers, their journeys, demographics or personal details. There are no privacy concerns associated with this dataset.

Data Dictionary

Variable Type Description
Year Integer Calendar year of observation (2018–2026)
Month Integer Month number (1–12)
Month_name Character Abbreviated month name (e.g. Jan, Feb)
Day_of_week Character Day of week (Monday through Sunday)
Day_type Character Normal Weekday, School Holiday Weekday, or Weekend
Mode Character PT mode: MetroTrain, Tram, MetroBus, RegionalTrain, RegionalBus
Pax_daily Numeric Daily average passenger boardings for that month, rounded to nearest 50

Dataset 2: ABS Consumer Price Index — Automotive Fuel

Source: Australian Bureau of Statistics (catalogue 6401.0)
Portal: abs.gov.au
URL: https://www.abs.gov.au/statistics/economy/price-indexes-and-inflation/consumer-price-index-australia/latest-release Licence: Creative Commons Attribution 4.0 Australia
Last accessed:

Type of data: The CPI is based on a sample survey, the ABS samples approximately 100,000 prices across retail outlets nationally each month. It is not a census of all prices. The automotive fuel series measures price change relative to a reference base period (September 2025 = 100).

Why this data is suitable: This data set provides monthly national fuel price data over the same period as the patronage data. It allows me to get some context with the patronage recovery story, specifically to examine whether periods of high fuel prices coincided with changes in PT patronage. Note that this data set is used contextually and not to establish causation (see limitations below).

Data limitations:

  • This is a national (Australia-wide) figure as Melbourne-specific fuel prices are not published separately by the ABS. The national index is used as a reasonable proxy given fuel prices are largely set by global oil markets and are broadly consistent across capital cities
  • The series was extracted programmatically from ABS CPI Table 3 (640103.xlsx), which contains 250+ columns covering all CPI categories. The automotive fuel series was identified and isolated in R using readxl and stringr
  • Care must be taken to avoid ecological fallacy as just because fuel prices and patronage change at the same time does not mean one caused the other. This dataset is used only to contextualise the structural nature of the patronage gap, not to claim causation

Ethical considerations: This dataset contains aggregate price index data only. There is no individual-level data and no privacy concerns.

# Find the automotive fuel column dynamically using "stringr"
# This avoids hardcoding the column number which could change 
# in future releases of the file

header_row <- as.character(fuel_raw[1, ])

auto_fuel_col <- which(
  str_detect(header_row, "Automotive fuel") &
  str_detect(header_row, "Index Numbers")
)
# Extract date and fuel index values
# Excel stores dates as numbers since 1899-12-30
# as.Date with origin converts them to proper R dates

fuel <- tibble(
  date = as.Date(
    as.numeric(fuel_raw[-(1:10), 1][[1]]),
    origin = "1899-12-30"
  ),
  fuel_index = as.numeric(fuel_raw[-(1:10), auto_fuel_col][[1]])
) |>
  filter(!is.na(fuel_index)) |>
  mutate(
    Year  = year(date),
    Month = month(date)
  )
